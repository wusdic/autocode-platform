## 六、两份评审的差异取舍（说明）

- **toolsets 写法**：评审 A 建议用 PyYAML 直接 patch `config.yaml`；本方案采用**更轻的 JSON 数组串 `config set`**——与真机已验证可行的 `docker_volumes` JSON 写法一致，且不引入 PyYAML 依赖。
- **模型默认值**：评审 A 默认 `glm-5.2`、评审 B 指出 z.ai 未必有 `glm-5.2`。**采纳 B**：默认 `glm-5.1`，`glm-5.2` 经环境变量启用。
- **ENV-02 预置空 `approved_versions.txt`**：评审 A 建议 `touch` 解死锁。**不采纳**——空文件在本插件逻辑里仍是"无批准版本"（按非空行判定），`touch` 不解锁且会误导；正确做法是设计阶段产出后再开闸（《03》Step 7）。dev-worker 本就应在设计批准后才开工，正常流程无死锁。
