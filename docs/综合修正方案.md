## 六、两份评审的差异取舍（说明）

- **toolsets 写法**：用 JSON 数组串 `config set`（`toolsets`/`disabled_toolsets` 是简单标量列表，`config set` 写 JSON 数组可正确解析）。**注意 `docker_volumes` 不同**：它是结构化列表，`config set` 会存成 YAML 字符串标量被 DockerEnvironment 丢弃，故 `docker_volumes` 必须用 pyyaml 直接写 config.yaml 为真正 YAML 列表（见 Bug-1）。两者写法差异已统一明确。
- **模型默认值**：按需求定为**默认 `glm-5.2` + DeepSeek**（两个大模型）。若 z.ai 账号无 glm-5.2，经 `ZAI_PRIMARY_MODEL`/`ZAI_SECONDARY_MODEL` 环境变量回退到 glm-5.1，无需动脚本。
- **ENV-02 预置空 `approved_versions.txt`**：评审 A 建议 `touch` 解死锁。**不采纳**——空文件在本插件逻辑里仍是"无批准版本"（按非空行判定），`touch` 不解锁且会误导；正确做法是设计阶段产出后再开闸（《03》Step 7）。dev-worker 本就应在设计批准后才开工，正常流程无死锁。
